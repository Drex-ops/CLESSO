\section{Background}

This document describes a neural network implementation of the CLESSO (Community-Level Species Similarity Observations) model. The core ecological framework -- estimating species richness ($\alpha$) and compositional turnover ($\beta$) from observation-pair data -- is identical to the TMB version described in the companion document. The key difference is that the parametric sub-models (linear terms, P-splines, I-splines) are replaced with neural networks, while preserving the ecological constraints that make the model interpretable.

The motivation for this variant is computational. The TMB implementation relies on a Laplace approximation over site-level random effects ($u_i$), which requires repeated sparse Cholesky factorisations of the joint precision matrix. For large numbers of sites ($>10^4$), this becomes prohibitively expensive -- often requiring days to converge. A neural network trained with mini-batch stochastic gradient descent eliminates the Laplace approximation entirely, reducing training from days to minutes while retaining (and potentially improving upon) the flexibility of the spline-based approach.

\section{Pair constructs and likelihood}

The data pipeline is shared with the TMB variant. We sample a set of observation pairs $P = P_{\text{within}} \cup P_{\text{between}}$ from species occurrence records. For each pair $p$ we record:

\begin{itemize}
    \item $i_p, j_p$: the sites involved
    \item $y_p = 0$ if the species identities match, $y_p = 1$ otherwise
    \item $w_p$: a weight that balances the four cells \{within-match, within-mismatch, between-match, between-mismatch\}
\end{itemize}

Under the equal-draw assumption from Hoskins et al.\ (2020), the match probability decomposes as:

\begin{equation}
    p_p = \begin{cases}
        \dfrac{1}{\alpha_i} & \text{if } i_p = j_p \text{ (within-site pair)} \\[10pt]
        S_{i,j} \cdot \dfrac{\alpha_i + \alpha_j}{2\,\alpha_i\,\alpha_j} & \text{if } i_p \neq j_p \text{ (between-site pair)}
    \end{cases}
\end{equation}

where $\alpha_i > 1$ is the species richness at site $i$ and $S_{i,j} = \exp(-\eta_{i,j}) \in [0,1]$ is the S\o rensen-type similarity derived from turnover $\eta_{i,j} \geq 0$.

The loss function is the weighted negative log-likelihood of the Bernoulli model plus a regularisation penalty:

\begin{equation}
    \mathcal{L} = -\frac{1}{\sum_p w_p}\sum_{p \in P} w_p \Big[(1-y_p)\log p_p + y_p \log(1-p_p)\Big] + \lambda_{\text{lb}} \cdot \mathcal{R}_{\alpha}
\end{equation}

This is identical to the TMB formulation. The difference lies entirely in \textit{how $\alpha_i$ and $\eta_{i,j}$ are parameterised}.

\section{Alpha sub-model: richness network}

In the TMB model, richness is parameterised as:

\begin{equation*}
    \log(\alpha_i - 1) = \theta_0 + \mathbf{z}_i^\top\bm{\theta} + \sum_k g_k(z_{k,i}^{\text{raw}}) + u_i
\end{equation*}

with linear terms, P-spline smooths $g_k(\cdot)$, and site-level random effects $u_i \sim \mathcal{N}(0, \sigma_u^2)$.

In the neural network variant, all of these components are replaced by a single multi-layer perceptron (MLP):

\begin{equation}
    \alpha_i = \text{softplus}\!\left(f_\alpha(\mathbf{z}_i; \bm{W}_\alpha)\right) + 1
\end{equation}

where $f_\alpha$ is a feedforward network with $L$ hidden layers:

\begin{equation}
    f_\alpha(\mathbf{z}) = \mathbf{W}^{(L+1)}\,\sigma\!\left(\mathbf{W}^{(L)}\,\sigma\!\left(\cdots\sigma\!\left(\mathbf{W}^{(1)}\,\mathbf{z} + \mathbf{b}^{(1)}\right)\cdots\right) + \mathbf{b}^{(L)}\right) + b^{(L+1)}
\end{equation}

Here $\sigma(\cdot)$ is a nonlinear activation, and $\mathbf{W}^{(\ell)}, \mathbf{b}^{(\ell)}$ are the weights and biases at layer $\ell$. The default architecture uses four hidden layers with dimensions $[256, 128, 64, 32]$ with SiLU (Sigmoid Linear Unit, $x\cdot\sigma(x)$) activation and dropout regularisation between layers. SiLU was chosen over ReLU for its smooth gradient landscape, which improves optimisation stability.

The output activation $\text{softplus}(\cdot) + 1$ guarantees $\alpha_i > 1$, matching the TMB constraint. The output bias is initialised so that $\text{softplus}(1.1) + 1 \approx 2.7$, providing a reasonable starting point for species richness. The key advantage is that the network simultaneously learns:
\begin{itemize}
    \item the linear effects captured by $\mathbf{z}^\top\bm{\theta}$ in the TMB model
    \item the nonlinear smooth terms $g_k(\cdot)$ captured by P-splines
    \item the site-level heterogeneity captured by random effects $u_i$
\end{itemize}

all as a single differentiable function, without requiring the Laplace approximation needed by the random effects.

\textbf{Note on random effects.} The flexibility of the hidden layers largely subsumes what $u_i$ captured in the TMB model. The P-spline basis with $K$ knots per covariate provides $\mathcal{O}(K)$ degrees of freedom; the network provides $\mathcal{O}(\sum_\ell d_\ell \cdot d_{\ell+1})$ parameters, which is typically much larger. If additional site-level variation is needed, a learned per-site embedding could be added, though this risks overfitting.

\subsection{Fourier positional encoding}

Raw longitude and latitude coordinates are poor features for neural networks because they impose a linear geometry on what is a spherical surface. To provide the alpha network with multi-scale spatial information, we optionally replace (or supplement) raw coordinates with Fourier positional encodings:

\begin{equation}
    \text{Fourier}_k(\text{lon}, \text{lat}) = \left[\sin\!\left(\frac{2\pi\,\text{lon}}{\lambda_k}\right),\; \cos\!\left(\frac{2\pi\,\text{lon}}{\lambda_k}\right),\; \sin\!\left(\frac{2\pi\,\text{lat}}{\lambda_k}\right),\; \cos\!\left(\frac{2\pi\,\text{lat}}{\lambda_k}\right)\right]
\end{equation}

with wavelengths $\lambda_k = \lambda_{\max} / 2^k$ for $k = 0, \ldots, N_f - 1$. With $\lambda_{\max} = 40^\circ$ and $N_f = 5$, this produces wavelengths $[40, 20, 10, 5, 2.5]$ degrees, spanning continental to regional scales. Each frequency contributes 4 features (sin/cos for each coordinate), so $N_f = 5$ adds 20 features to $\mathbf{z}_i$.

This encoding allows the network to learn spatially structured richness patterns at multiple scales without requiring an explicit spatial random effect. When raw coordinates are excluded (\texttt{exclude\_coords\_from\_alpha=True}), the Fourier features are the sole source of geographic information.

\subsection{Alpha lower-bound penalty}

As in the TMB model, a soft one-sided penalty discourages $\alpha_i < S_{\text{obs},i}$ (the observed species count):

\begin{equation}
    \mathcal{R}_\alpha = \frac{1}{|B|}\sum_{i \in B}\left[\frac{\log(1 + e^{\kappa(S_{\text{obs},i} - \alpha_i)})}{\kappa}\right]^2
\end{equation}

where $\kappa = 10$ sharpens the hinge boundary and $B$ is the current mini-batch. This is the same softplus-squared penalty used in the TMB model, applied to both $\alpha_i$ and $\alpha_j$ in each pair by indexing into the observed-richness vector.

\subsection{Effort/detectability decomposition}
\label{sec:effort-decomposition}

Under the equal-draw assumption, $\alpha$ represents an \emph{effective richness under the recording process} rather than true species richness. Consequently, spatial variation in sampling effort, observer intensity, institutional proximity, and road access can all inflate or deflate $\alpha$ estimates, creating artefactual structure that may be absorbed by the turnover model if not accounted for explicitly. To separate the observation-process signal from the ecological signal, the alpha model is decomposed into two additive components on the logit scale:

\begin{equation}
    \alpha_i = \text{softplus}\!\Big(\underbrace{f_\alpha(\mathbf{z}_i;\, \bm{W}_\alpha)}_{\text{environment logit}} + \underbrace{f_\omega(\mathbf{w}_i;\, \bm{W}_\omega)}_{\text{effort logit}}\Big) + 1
\end{equation}

where $\mathbf{z}_i$ is the vector of environmental and substrate covariates (as before) and $\mathbf{w}_i$ is a separate vector of effort/detectability covariates at site~$i$. The two sub-networks are:

\begin{description}
    \item[EnvNet] $f_\alpha(\mathbf{z})$: the existing alpha MLP (Section~3), unchanged in architecture. Its output is now interpreted as the pre-activation \emph{environment logit}.
    \item[EffortNet] $f_\omega(\mathbf{w})$: a smaller MLP (default hidden layers $[64, 32]$, SiLU activation, dropout) mapping effort covariates to a scalar \emph{effort logit}. The output bias is initialised to zero so that the effort contribution is neutral at the start of training, i.e.\ $f_\omega(\mathbf{w}) \approx 0$ initially.
\end{description}

The additive decomposition has several desirable properties:
\begin{enumerate}
    \item \textbf{Clean separation at prediction time.} To predict true ecological richness (uncontaminated by sampling artefacts), we simply zero-out the effort logit: $\alpha_i^{\text{env}} = \text{softplus}(f_\alpha(\mathbf{z}_i)) + 1$. No EffortNet evaluation is needed.
    \item \textbf{Diagnostic mappability.} The effort logit $f_\omega(\mathbf{w}_i)$ can be mapped spatially to visualise where sampling effort inflates or deflates apparent richness.
    \item \textbf{Neutral initialisation.} Because $f_\omega$ starts near zero, the model begins training as if effort covariates were absent. The effort signal is learned only if it improves the likelihood.
    \item \textbf{Backward compatibility.} When no effort covariates are supplied ($K_\omega = 0$), the EffortNet is not instantiated and the model reduces to the original architecture.
\end{enumerate}

\paragraph{Effort covariates.}
Six raster-derived effort layers are extracted at each site location:

\begin{center}
\begin{tabular}{ll}
    \toprule
    Covariate & Description \\
    \midrule
    \texttt{ala\_record\_count} & Total ALA occurrence records per grid cell \\
    \texttt{ala\_record\_smoothed} & Gaussian-smoothed record density \\
    \texttt{dist\_to\_nearest\_institution} & Distance (km) to nearest collection/museum \\
    \texttt{hub\_influence\_unweighted} & Gravity-weighted influence of all institutions \\
    \texttt{hub\_influence\_ecology\_weighted} & As above, weighted by ecology-relevant collections \\
    \texttt{road\_density\_km\_per\_km2} & Road network density (km road per km$^2$) \\
    \bottomrule
\end{tabular}
\end{center}

All layers share the same 0.05$^\circ$ grid as the environmental covariates (670$\times$813 pixels). Values are standardised (zero-mean, unit-variance) before input to EffortNet.

\paragraph{Regularisation.}
An optional $L_2$ penalty on EffortNet parameters (controlled by \texttt{effort\_penalty}) can be used to shrink the effort component toward zero, encouraging the model to explain as much variation as possible through the environmental pathway. The penalty is added to the total loss:
\begin{equation}
    \mathcal{L}_{\text{effort}} = \lambda_\omega \sum_{\theta \in \bm{W}_\omega} \theta^2
\end{equation}

\paragraph{Interaction with training modes.}
The effort decomposition is active from the start of training in all modes (joint, two-stage, cyclic, cyclic fine-tune). During cyclic fine-tuning Phase~2, EffortNet weights are transferred from Phase~1 alongside the existing alpha architecture expansion.

\section{Beta sub-model: monotone turnover network}

\subsection{TMB approach}

In the TMB model, turnover is parameterised using I-spline basis functions with non-negative coefficients:

\begin{equation*}
    \eta_{i,j} = \exp(\eta_0^{\text{raw}}) + \sum_{l=1}^{m}\sum_{s=1}^{S} \exp(\beta_{l,s}^{\text{raw}}) \cdot \left|I_s(x_{l,i}) - I_s(x_{l,j})\right|
\end{equation*}

where $I_s(\cdot)$ are monotonically non-decreasing I-spline bases and the exponentiation of the raw coefficients ensures $\beta_{l,s} \geq 0$. The composition of non-negative coefficients with monotone bases guarantees that $\eta_{i,j}$ is non-decreasing in each $|x_{l,i} - x_{l,j}|$.

\subsection{Shared building block: MonotoneLinear layers}

All beta network variants rely on a \textbf{MonotoneLinear} layer, which is a standard linear layer with the weight matrix constrained to be element-wise non-negative via a softplus transform on unconstrained parameters:

\begin{equation}
    \tilde{\mathbf{W}}^{(\ell)} = \text{softplus}(\mathbf{W}^{(\ell)}_{\text{raw}}) \geq 0
\end{equation}

The unconstrained $\mathbf{W}^{(\ell)}_{\text{raw}}$ are the learnable parameters; the optimiser updates them freely while the softplus transform guarantees non-negative effective weights. The bias terms $\mathbf{b}^{(\ell)}$ are unconstrained.

\textbf{Monotonicity proof.} Consider the composition at any layer $\ell$:
\begin{enumerate}
    \item The input $\mathbf{x}^{(\ell-1)} \geq \mathbf{0}$ (base case: $\bm{\Delta} \geq 0$)
    \item The linear map $\tilde{\mathbf{W}}^{(\ell)}\mathbf{x}^{(\ell-1)} + \mathbf{b}^{(\ell)}$ is non-decreasing in $\mathbf{x}^{(\ell-1)}$ because $\tilde{\mathbf{W}}^{(\ell)} \geq 0$
    \item ReLU (or softplus) is monotone non-decreasing, so $\sigma(\cdot)$ preserves the ordering
    \item By induction, the full composition is monotone non-decreasing in each input dimension
\end{enumerate}

Critically, \textbf{$L_2$ weight decay must not be applied to $\mathbf{W}_{\text{raw}}$}. Standard $L_2$ regularisation penalises the raw parameters toward zero, but $\text{softplus}(0) = \ln 2 \approx 0.693$ for every weight, which drives all effective weights to the same value and causes \textit{dimensional collapse} --- all environmental dimensions receive identical weight regardless of their ecological importance.

\subsection{Deep monotone network (\texttt{beta\_type=``deep''})}

The simplest approach replaces the I-spline transform with a deep monotone MLP:

\begin{equation}
    \eta_{i,j} = \text{softplus}\!\left(h_\beta(\bm{\Delta}_{i,j}; \bm{W}_\beta)\right)
\end{equation}

where $\bm{\Delta}_{i,j} = \left[|\tilde{x}_{1,i} - \tilde{x}_{1,j}|, \ldots, |\tilde{x}_{m,i} - \tilde{x}_{m,j}|\right] \geq \mathbf{0}$ is the vector of standardised environmental distances and $h_\beta$ is a monotone MLP built from MonotoneLinear layers with ReLU activations.

This directly generalises the TMB approach: the I-spline + non-negative coefficient structure \textit{is} a single-layer monotone network with a specific (spline) feature transform. The multi-layer version is a universal approximator for monotone functions (Daniels \& Velikova, 2010). However, the deep variant is prone to dimensional collapse (all weights converging under weight decay) and is not the recommended architecture.

\subsection{Additive monotone network (\texttt{beta\_type=``additive''})}
\label{sec:additive-beta}

The additive beta network is the direct neural analog of GDM's additive I-spline structure. Turnover decomposes as a sum of independent per-dimension monotone functions:

\begin{equation}
    \eta_{i,j} = \sum_{k=1}^{m} f_k\!\left(|\tilde{x}_{k,i} - \tilde{x}_{k,j}|\right)
\end{equation}

where each $f_k: \mathbb{R}_{\geq 0} \to \mathbb{R}$ is an independent monotone network with $n$ hidden units (default $n = 32$):

\begin{equation}
    g_k(d) = \mathbf{w}_2^\top\,\text{softplus}\!\left(\tilde{\mathbf{w}}_1\, d + \mathbf{b}_1\right), \qquad f_k(d) = g_k(d) - g_k(0)
\end{equation}

Here $\tilde{\mathbf{w}}_1 = \text{softplus}(\mathbf{w}_{1,\text{raw}}) \geq 0$ and $\mathbf{w}_2 = \text{softplus}(\mathbf{w}_{2,\text{raw}}) \geq 0$ are non-negative weight vectors. The hidden activation is \textbf{softplus} rather than ReLU: softplus has no dead zone (always nonzero gradient), giving naturally gradual onset of turnover rather than the hard on/off threshold behaviour of ReLU. Softplus is still monotone non-decreasing, preserving the monotonicity guarantee.

The \textbf{baseline subtraction} $f_k(d) = g_k(d) - g_k(0)$ ensures that $f_k(0) = 0$ exactly, so zero environmental distance contributes zero turnover. Without this, the network biases would introduce a nonzero $\eta$ even for identical sites.

The biases $\mathbf{b}_1$ in the first layer play the role of knot locations: they are initialised uniformly from $-3\sigma$ to $0\sigma$ of the standardised environmental distance, spreading the response onset across the observed range. The second-layer weights are initialised very small (at $\text{softplus}(-3) \approx 0.05$ effective weight) so that each dimension initially contributes $\approx 0.3$ at maximum distance, giving a total initial $\eta \approx 5$ across all dimensions.

The final output is clamped: $\eta_{i,j} = \text{clamp}\!\left(\sum_k f_k, 0, 10\right)$. The upper bound of 10 corresponds to $S = e^{-10} \approx 4.5 \times 10^{-5}$, effectively complete dissimilarity. Without this clamp, the model can push $\eta \to \infty$ for all between-site pairs, learning the degenerate solution $p \approx 0$ everywhere.

\subsection{Factored deep monotone network (\texttt{beta\_type=``factored''})}
\label{sec:factored-beta}

The factored architecture combines the per-dimension specialisation of the additive network with learned cross-dimension interactions:

\begin{equation}
    \eta_{i,j} = \underbrace{\sum_{k=1}^{m} s_k\!\left(|\Delta x_k|\right)}_{\text{additive component}} + \underbrace{\phi\!\left(\mathbf{h}_1, \ldots, \mathbf{h}_m\right)}_{\text{interaction component}}
\end{equation}

The architecture proceeds in three stages:
\begin{enumerate}
    \item \textbf{Per-dimension monotone encoders}: Each $|\Delta x_k|$ (scalar) is mapped through a private multi-layer monotone encoder to a hidden representation $\mathbf{h}_k \in \mathbb{R}^{d_{\text{enc}}}_{\geq 0}$ (default $d_{\text{enc}} = 16$, depth 2). The final activation is ReLU, ensuring $\mathbf{h}_k \geq 0$ element-wise.
    
    \item \textbf{Per-dimension scalar projections}: Each $\mathbf{h}_k$ is projected to a scalar $s_k = \mathbf{w}_k^\top \mathbf{h}_k$ via a MonotoneLinear($d_{\text{enc}}, 1$) layer, then baseline-subtracted: $s_k(\Delta x_k) - s_k(0)$. The sum $\sum_k s_k$ forms the additive component.
    
    \item \textbf{Monotone interaction network}: The concatenation $[\mathbf{h}_1; \ldots; \mathbf{h}_m] \in \mathbb{R}^{m \cdot d_{\text{enc}}}$ is fed through a shared monotone MLP (MonotoneLinear layers with softplus activation) producing a scalar interaction term. This is also baseline-subtracted: $\phi(\mathbf{h}) - \phi(\mathbf{0})$.
\end{enumerate}

\textbf{Monotonicity guarantee}: Each encoder is monotone in its input (non-negative weights + monotone activations). The interaction network has non-negative weights (MonotoneLinear) and takes non-negative inputs ($\mathbf{h}_k \geq 0$ via ReLU), so it is monotone in each $\mathbf{h}_k$. Since the composition of monotone functions is monotone, the full network is monotone in each $|\Delta x_k|$. Dropout is disabled throughout the factored network because baseline subtraction requires deterministic forward passes.

The interaction term allows the model to capture effects such as: the impact of temperature distance on turnover depends on the magnitude of precipitation distance. The additive network (Section~\ref{sec:additive-beta}) cannot represent such interactions.

\subsection{Geographic distance as a beta input}

Optionally, the beta network can include geographic (Haversine) distance as an additional input dimension:

\begin{equation}
    \bm{\Delta}_{i,j} = \left[|\tilde{x}_{1,i} - \tilde{x}_{1,j}|, \ldots, |\tilde{x}_{m,i} - \tilde{x}_{m,j}|, \; \frac{d_{\text{geo}}(i,j)}{\sigma_{\text{geo}}}\right]
\end{equation}

where $d_{\text{geo}}(i,j)$ is the great-circle distance in kilometres and $\sigma_{\text{geo}}$ is estimated from random site pairs. This additional dimension gets its own monotone sub-network (additive or encoded) and allows the model to capture \textit{distance decay} --- the empirical observation that community similarity decreases with geographic distance independent of environmental differences (e.g.\ dispersal limitation).

The geographic distance feature is typically introduced in Phase~2 of the cyclic fine-tuning procedure (Section~\ref{sec:cyclic-finetune}), after the environmental signal has been learned.

The additive network is the default (\texttt{beta\_type=``additive''}) as it most closely mirrors the GDM/TMB structure and produces directly interpretable per-variable response curves. The factored variant is available when cross-variable interactions are suspected.

\section{Optimisation}

The TMB model is fitted via maximum marginal likelihood using \texttt{nlminb}, with the Laplace approximation integrating over the random effects $\mathbf{u}$. Each function evaluation requires a sparse Cholesky factorisation of the joint precision matrix, which is $\mathcal{O}(n)$ in the number of sites but with a large constant.

The neural network is fitted via mini-batch stochastic gradient descent using Adam (Kingma \& Ba, 2015). Each parameter update costs $\mathcal{O}(B \cdot D)$ where $B$ is the batch size and $D$ is the total number of network parameters. There is no Hessian or Laplace approximation.

\begin{center}
\begin{tabular}{l|l|l}
    & TMB & Neural network \\
    \hline
    Optimiser & \texttt{nlminb} (quasi-Newton) & Adam (adaptive SGD) \\
    Random effects & Laplace approximation & None (absorbed by network) \\
    Per-iteration cost & Sparse Cholesky ($\sim$ seconds) & Matrix multiplies ($\sim$ milliseconds) \\
    Typical runtime & Hours to days & Minutes \\
    Hardware & CPU only & CPU, GPU, or Apple MPS \\
\end{tabular}
\end{center}

Training details:
\begin{itemize}
    \item \textbf{Learning rate scheduling}: Cosine annealing with warm restarts ($T_0 = 30$ epochs, $T_{\text{mult}} = 2$), giving cycle lengths of 30, 60, 120,\ldots\ epochs. The minimum LR is $10^{-6}$.
    \item \textbf{Early stopping}: halt training if validation loss does not improve for a configurable number of epochs (default 40).
    \item \textbf{Gradient clipping}: max norm of 5.0 on all parameters to prevent exploding gradients.
    \item \textbf{Weight decay}: $L_2$ penalty of $10^{-5}$ on alpha parameters and beta bias parameters. \textit{No weight decay} on beta \texttt{weight\_raw} parameters (see Section~\ref{sec:additive-beta}).
    \item \textbf{Dropout}: applied between hidden layers (default rate 0.1) to reduce overfitting.
    \item \textbf{Batch size}: 8192 pairs per mini-batch (default).
\end{itemize}

\subsection{Gradient imbalance between alpha and beta}
\label{sec:grad-imbalance}

A fundamental challenge in jointly training the alpha and beta sub-models is a severe gradient imbalance. The match probability for between-site pairs is:
\begin{equation*}
    p_{\text{between}} = \underbrace{e^{-\eta}}_{S} \cdot \frac{\alpha_i + \alpha_j}{2\,\alpha_i\,\alpha_j}
\end{equation*}

Differentiating the log-likelihood with respect to $\eta$ introduces a factor of $\partial p / \partial \eta = -p$, so beta gradients are attenuated by a factor of approximately $p \sim 1/\alpha$. For realistic richness values ($\alpha \approx 3$--$100$), beta gradients can be $100$--$1000\times$ smaller than alpha gradients, effectively starving the turnover network of learning signal.

Three mechanisms address this imbalance:

\begin{enumerate}
    \item \textbf{Beta LR multiplier}: The beta network receives a higher base learning rate than alpha (default $10\times$), implemented via separate Adam parameter groups.
    
    \item \textbf{Gradient amplification hooks}: PyTorch backward hooks on beta parameters scale their gradients by a configurable factor (default $100\times$ in joint training, $10\times$ in two-stage/cyclic modes). This is applied \textit{before} gradient clipping.
    
    \item \textbf{Importance weighting} (in Stage~2 / beta phases): Match pairs ($y=0$) are reweighted by $1/\alpha_h$ where $\alpha_h = 2\alpha_i\alpha_j/(\alpha_i + \alpha_j)$ is the harmonic mean richness. This balances the gradient contributions of match vs.\ mismatch pairs:
    \begin{equation}
        \mathcal{L}_{\text{beta}} = \frac{\sum_p \tilde{w}_p \cdot \text{BCE}(y_p, p_p)}{\sum_p \tilde{w}_p}, \qquad \tilde{w}_p = \begin{cases} 1/\alpha_h & \text{if } y_p = 0 \text{ (match)} \\ 1 & \text{if } y_p = 1 \text{ (mismatch)} \end{cases}
    \end{equation}
\end{enumerate}

\subsection{Training modes}
\label{sec:training-modes}

Four training modes are implemented, addressing the alpha--beta gradient imbalance with increasing sophistication:

\subsubsection{Joint training (\texttt{mode=``joint''})}

All parameters are optimised simultaneously on all pairs. This is the simplest mode but is prone to \textit{beta collapse}: alpha inflates to match sampled pair frequencies rather than true richness, making $\partial p / \partial \eta \approx 0$ and leaving beta unable to learn. Gradient amplification and the beta LR multiplier partially mitigate this, but joint training is not recommended for production use.

\subsubsection{Two-stage training (\texttt{mode=``two\_stage''})}
\label{sec:two-stage}

Mirrors the original CLESSO two-stage approach (\texttt{clesso\_alpha.cpp} $\to$ \texttt{clesso\_beta\_fixAlpha.cpp}):

\begin{description}
    \item[Stage 1:] Freeze beta, train alpha on \textit{within-site pairs only} with BCE + lower-bound penalty. Alpha converges to realistic richness values ($\alpha \approx S_{\text{obs}}$).
    \item[Stage 2:] Freeze alpha, train beta on \textit{between-site pairs only} with importance-weighted BCE. Beta receives uncontested, balanced gradients and learns non-trivial turnover response curves.
\end{description}

Each stage uses its own optimizer (Adam with cosine annealing) and early stopping.

\subsubsection{Cyclic block-coordinate descent (\texttt{mode=``cyclic''})}
\label{sec:cyclic}

Mirrors \texttt{clesso\_iterative.R}: alternating block-coordinate optimisation with damped step sizes:

\begin{center}
\textbf{for} cycle $= 1, \ldots, C_{\max}$: \quad
(A) freeze $\beta$, train $\alpha$ for $N_\alpha$ epochs \quad
(B) freeze $\alpha$, train $\beta$ for $N_\beta$ epochs
\end{center}

Default settings use $N_\alpha = 5$ and $N_\beta = 10$ (small alpha steps, 2$\times$ beta steps), keeping alpha changes incremental so beta can adapt without oscillation. Convergence is declared when the relative change in best beta validation loss between cycles falls below a threshold ($10^{-4}$).

Key differences from two-stage:
\begin{itemize}
    \item Alpha and beta \textit{see each other's updated parameters repeatedly}, allowing co-adaptation.
    \item Alpha can re-adjust after beta has learnt turnover (e.g.\ correcting inflated estimates).
    \item Persistent optimiser state (momentum) is maintained across cycles.
    \item Step-based LR decay (halve every 200 global epochs) replaces per-stage cosine annealing.
\end{itemize}

Cyclic training is the default mode (\texttt{training\_mode=``cyclic''}).

\subsubsection{Cyclic + geographic fine-tuning (\texttt{mode=``cyclic\_finetune''})}
\label{sec:cyclic-finetune}

A two-phase procedure that separates environmental and geographic signal learning:

\begin{description}
    \item[Phase 1:] Damped cyclic training on \textit{environmental and substrate covariates only} (no Fourier features, no geographic distance). This learns the core ecological signal without geographic confounding.
    
    \item[Phase 2:] Expand the model architecture to accommodate new input dimensions:
    \begin{itemize}
        \item Alpha: add Fourier positional encoding features (zero-initialised weights so Phase~1 predictions are initially unchanged).
        \item Beta: add a geographic distance dimension (new per-dimension monotone sub-network with fresh random initialisation).
    \end{itemize}
    Fine-tune with a smaller learning rate. Optionally, Phase~1 parameters can be frozen (\texttt{finetune\_freeze\_existing=True}) so only the new geographic parameters are trained.
\end{description}

This mirrors the intuition from spatial ecology that environmental variables should explain most compositional turnover, with geographic distance capturing residual dispersal limitation.

\subsection{Hard-pair mining with importance correction}
\label{sec:hard-mining}

The pairwise likelihood in equation~(2) is defined with respect to a \emph{target pair distribution} over the sampled pair set~$P$, represented by the existing cell-balancing weights~$w_p$. Na\"ive hard-pair mining changes this training distribution by over-sampling difficult pairs, which can improve optimisation speed but distort the resulting fitted probabilities. To retain the original probabilistic target while focusing optimisation on informative pairs, hard-pair mining is introduced as a \emph{proposal distribution} with importance correction.

\paragraph{Target and proposal distributions.}
Let $\pi(p)$ denote the target sampling probability for pair~$p$, corresponding to the original pair-construction design, and let $q(p)$ denote the actual probability that pair~$p$ is selected into a mini-batch under the mining scheme. The training loss uses an importance-corrected weight:
\begin{equation}
    w_p^\star = w_p \cdot \frac{\pi(p)}{q(p)}
\end{equation}

In practice, three layers of importance correction are composed multiplicatively. For a between-site pair in a mined mini-batch, the effective weight is:
\begin{equation}
    w_p^{\text{eff}} = \underbrace{w_p\vphantom{\frac{1}{\alpha_h}}}_{\substack{\text{population}\\\text{proportion}}} \;\cdot\; \underbrace{\frac{1}{\alpha_h}}_{\substack{\text{gradient}\\\text{balance}}} \;\cdot\; \underbrace{\frac{\pi(p)}{q(p)}}_{\substack{\text{mining}\\\text{correction}}}
\end{equation}
where $\alpha_h = 2\alpha_i\alpha_j/(\alpha_i + \alpha_j)$ is the harmonic mean richness from Section~\ref{sec:grad-imbalance}. The first factor corrects for the 50/50 match/mismatch sampling design, the second balances the gradient attenuation from Section~\ref{sec:grad-imbalance}, and the third corrects for the non-uniform mini-batch sampling introduced by mining. When $q(p) = \pi(p)$, the mining correction is unity and the loss reduces to the original importance-weighted likelihood.

The mini-batch loss is:
\begin{equation}
    \widehat{\mathcal{L}}_B
    =
    -\frac{1}{\sum_{p \in B} w_p^{\text{eff}}}
    \sum_{p \in B}
    w_p^{\text{eff}}
    \Big[(1-y_p)\log p_p + y_p \log(1-p_p)\Big]
    + \lambda_{\text{lb}} \cdot \mathcal{R}_\alpha
\end{equation}

\paragraph{Coarse pair strata.}
Hard-pair mining is applied only to \emph{between-site} pairs during beta training phases, since within-site pairs anchor the alpha model and have a fundamentally different loss surface ($p = 1/\alpha$, no $\eta$ dependence). Between-site pairs are divided into two coarse cells:
\[
c(p) \in \{\text{between-match},\; \text{between-mismatch}\}
\]
Each cell receives equal target mass, so the target distribution is:
\begin{equation}
    \pi(p) = \Pr(c(p)) \cdot \frac{1}{N_{c(p)}} = \frac{1}{2} \cdot \frac{1}{N_{c(p)}}
\end{equation}
where $N_c$ is the number of training pairs in cell~$c$.

\paragraph{Difficulty strata.}
For each between-site pair~$p$, a hardness score is computed using the current per-pair negative log-likelihood:
\begin{equation}
    h_p = -\Big[(1-y_p)\log p_p + y_p \log(1-p_p)\Big]
\end{equation}
Within each coarse cell, pairs are ranked by $h_p$ and partitioned into $D$ difficulty bins (default $D = 3$: easy, medium, hard) using quantile boundaries. Let $d(p)$ denote the difficulty bin for pair~$p$, and $N_{c,d}$ the number of pairs in cell~$c$ and bin~$d$.

Using quantile-based bins rather than absolute NLL thresholds avoids a subtle issue: pairs near the $\eta = 10$ clamp boundary (Section~\ref{sec:additive-beta}) have irreducibly high NLL due to the modelling constraint, not because they are informative. Quantile ranking within each cell ensures these pairs are distributed across bins rather than dominating the ``hard'' bin.

\paragraph{Mixture proposal.}
The proposal distribution for between-site pairs is a mixture:
\begin{equation}
    q(p) = (1-\lambda_{\text{hm}})\,\pi(p) + \lambda_{\text{hm}}\,h^\star\!(p)
\end{equation}
where $\lambda_{\text{hm}} \in [0,1]$ controls the mining strength and $h^\star\!(p)$ is the hard-pair proposal:
\begin{equation}
    h^\star\!(p)
    =
    \Pr(c(p)) \cdot \Pr(d(p)\mid c(p)) \cdot \frac{1}{N_{c(p),d(p)}}
\end{equation}
The conditional bin probabilities $\Pr(d \mid c)$ assign extra mass to difficult bins (default: $[0.2, 0.3, 0.5]$ for easy/medium/hard). At $\lambda_{\text{hm}} = 0$ (default), the proposal equals the target and all mining code is a no-op.

\paragraph{Variance control.}
The importance ratio $\pi(p)/q(p)$ can become large when easy pairs are drawn from the hard-biased proposal. To prevent unstable gradients, the ratio is truncated:
\begin{equation}
    \frac{\pi(p)}{q(p)}
    \;\leftarrow\;
    \min\!\left(\frac{\pi(p)}{q(p)},\; a_{\max}\right)
\end{equation}
with $a_{\max} = 5$ by default. This introduces a small bias in exchange for substantially reduced gradient variance. The truncation interacts with the existing $1/\alpha_h$ importance weighting: high-$\alpha$ match pairs from easy bins receive the product of both corrections, so the truncation cap prevents doubly extreme weights.

\paragraph{Practical training scheme.}
Training proceeds in two phases:
\begin{enumerate}
    \item \textbf{Warm-up.} For the first few cycles (default 3), mini-batches are sampled from the target distribution only ($\lambda_{\text{hm}} = 0$). This avoids using unstable model-based hardness scores early in training when the beta network has not yet learned meaningful turnover.
    \item \textbf{Mining phase.} At the start of each subsequent beta training phase (Phase~B of cyclic training, or each epoch in two-stage mode), a forward pass over all between-site training pairs computes hardness scores~$h_p$, refreshes the difficulty bins, and rebuilds the proposal distribution~$q(p)$. Scores can optionally be refreshed mid-phase (default: every beta epoch) to track the evolving loss landscape.
\end{enumerate}

The scoring forward pass is inexpensive --- it amounts to one epoch of inference over the training pairs with no gradient computation --- and is easily amortised over the subsequent training epochs.

\paragraph{Interpretation.}
This scheme preserves the original probabilistic target of the weighted Bernoulli likelihood while allowing optimisation to focus more heavily on informative between-site pairs. In effect, hard-pair mining is treated as a \emph{sampling design for stochastic optimisation} rather than a change to the statistical model itself. The fitted match probabilities remain calibrated with respect to the original pair distribution, which is critical because downstream predictions (beta-diversity indices, compositional similarity maps) rely on these probabilities being interpretable rather than merely ordinal.

\section{Relationship to TMB model}

The neural network variant is not a different model in the ecological sense. The likelihood (equation 2), the match probability decomposition (equation 1), and the pair sampling strategy are all identical. What changes is the function class used to map covariates to $\alpha$ and environmental distances to $\eta$:

\begin{center}
\begin{tabular}{l|l|l}
    Component & TMB & Neural network \\
    \hline
    $\alpha_i$ & Linear + P-spline + random effect & MLP (SiLU) with softplus output \\
    Effort separation & Not modelled & Additive EffortNet decomposition \\
    $\eta_{i,j}$ & I-spline + non-negative coefficients & Additive/factored monotone network \\
    $p_p$ & Same formula & Same formula \\
    $\mathcal{L}$ & Same weighted Bernoulli NLL & Same (+ importance weighting for $\beta$) \\
    $\mathcal{R}_\alpha$ & Same softplus$^2$ lower-bound penalty & Same softplus$^2$ lower-bound penalty \\
    Training & \texttt{nlminb}, two-stage, iterative & Adam, two-stage/cyclic block-coordinate \\
    Geography & I-spline on geo distance & Fourier encoding ($\alpha$) + haversine dim ($\beta$) \\
\end{tabular}
\end{center}

The neural network can be viewed as replacing the manually specified basis functions (B-splines, I-splines) with learned feature representations, and replacing the Gaussian random effects with implicit regularisation through dropout and weight decay. The additive beta network (default) preserves the per-variable interpretability of GDM while gaining the flexibility of learned (rather than pre-specified) knot placements.

\section{Post-training: diagnostics and spatial prediction}

After training, the pipeline automatically runs three post-processing steps:

\subsection{Diagnostics}

A comprehensive diagnostic PDF (mirroring the R script \texttt{clesso\_diagnostics.R}) is generated, containing:
\begin{itemize}
    \item Training curves (loss, accuracy, alpha statistics, learning rate schedule)
    \item Residual analysis (within-site and between-site)
    \item Observed vs.\ predicted match probability
    \item Classification metrics (AUC, confusion matrix)
    \item Spatial map of fitted $\alpha$ (richness)
    \item $\alpha$ vs.\ observed species richness comparison
    \item Beta turnover response curves per environmental dimension (analogous to I-spline partial functions)
    \item Similarity decay curves $S = \exp(-\eta)$
    \item Alpha partial-dependence plots (site covariates $\to$ richness)
    \item Variable importance bar chart
\end{itemize}

\subsection{Alpha surface}

Species richness is predicted across all of Australia using the trained alpha network applied to a gridded covariate stack (0.05$^\circ$ resolution, 670$\times$813 cells). The covariates include 6 substrate PCA bands, 9 AWAP climate variables (matching training columns), and optionally Fourier positional features.

When the effort decomposition is active (Section~\ref{sec:effort-decomposition}), three alpha surfaces are produced:
\begin{enumerate}
    \item \textbf{Environment-only} ($\alpha^{\text{env}}$): the primary prediction surface, computed as $\text{softplus}(f_\alpha(\mathbf{z})) + 1$ with the effort logit zeroed. This represents estimated true richness uncontaminated by sampling artefacts and is saved as the main GeoTIFF.
    \item \textbf{Full} ($\alpha^{\text{full}}$): the combined prediction $\text{softplus}(f_\alpha(\mathbf{z}) + f_\omega(\mathbf{w})) + 1$, representing apparent richness as influenced by both environment and observation process. Requires effort raster layers at the same grid resolution.
    \item \textbf{Effort logit} ($f_\omega(\mathbf{w})$): the raw effort network output, mapped spatially to visualise where sampling effort inflates (positive values) or deflates (negative values) apparent richness relative to the environmental baseline.
\end{enumerate}
All three are saved as separate GeoTIFF files.

\subsection{Beta surface (community composition map)}

A spatial map of biological community composition is generated using Landmark MDS:
\begin{enumerate}
    \item Select $k$ landmark pixels via spatial $k$-means (default $k = 500$)
    \item Compute pairwise $\eta$ between all landmark pairs using the trained beta network ($k \times k$ matrix)
    \item Compute $\eta$ from every pixel to each landmark ($n \times k$ matrix)
    \item Convert to dissimilarity: $D = 1 - \exp(-\eta)$
    \item Classical MDS on the landmark dissimilarity matrix + Nystr\"om extension for all pixels
    \item Map 3 MDS dimensions to RGB channels
\end{enumerate}

The result is a 3-band GeoTIFF where similar colours indicate similar predicted community composition.

\section{Identifiability considerations and practical safeguards}

A central challenge in the ObsPair formulation is that the between-site match probability depends on both richness and turnover:
\begin{equation}
    p_{ij}
    =
    S_{ij}\cdot \frac{\alpha_i+\alpha_j}{2\alpha_i\alpha_j},
    \qquad
    S_{ij} = \exp(-\eta_{ij}).
\end{equation}
Consequently, a low between-site match probability can be explained either by low similarity (\(\beta\)-driven turnover) or by high effective richness at one or both sites (\(\alpha\)-driven dilution of the chance of a match). This creates a genuine identifiability problem unless the two components are anchored by different sources of information or are constrained to play distinct structural roles.

\subsection{Primary source of identifiability}

The strongest direct information for \(\alpha_i\) comes from within-site pairs, for which
\begin{equation}
    p_{ii} = \frac{1}{\alpha_i}.
\end{equation}
This implies that within-site pairs are the main empirical anchor for the richness model, whereas between-site pairs are responsible for learning turnover conditional on that baseline. In practice, identifiability improves when the fitting procedure preserves this division of labour. Useful strategies include:
\begin{itemize}
    \item assigning sufficient weight to within-site pairs during training,
    \item pretraining the \(\alpha\) model on within-site pairs before joint optimisation,
    \item freezing or partially damping \(\alpha\) while the \(\beta\) model is first introduced,
    \item and using auxiliary constraints on \(\alpha\) based on observed richness or sample-completeness information when available.
\end{itemize}

\subsection{Constraining the turnover baseline}

A further safeguard is to prevent the \(\beta\) model from learning a free baseline reduction in match probability that competes with \(\alpha\). In particular, if turnover is parameterised as
\begin{equation}
    \eta_{ij} = \eta_0 + f(\bm{\Delta}_{ij}),
\end{equation}
then the intercept \(\eta_0\) can absorb residual mismatch even when environmental distance is zero, weakening the separation between richness and turnover. A cleaner formulation enforces
\begin{equation}
    \eta(\mathbf 0) = 0,
\end{equation}
so that turnover only explains additional decay in similarity with increasing environmental separation. This is consistent with the ecological interpretation that identical environmental conditions should not imply positive turnover in the modeled distance component.

\subsection{Controlling flexibility asymmetrically}

Because \(\alpha\) and \(\beta\) both influence between-site match probabilities, identifiability is sensitive to their relative flexibility. In practice, the \(\alpha\) model is often the more dangerous absorber of residual structure, because it contributes to both within-site and between-site probabilities and can mimic unmodeled heterogeneity. It is therefore often beneficial to keep the \(\alpha\) model somewhat more conservative than the \(\beta\) model, for example by:
\begin{itemize}
    \item using a smaller network for \(\alpha\) than for \(\beta\),
    \item applying stronger weight decay or dropout to \(\alpha\),
    \item shrinking interaction terms more strongly than main effects,
    \item and restricting any spatial residual term in \(\alpha\) to be broad-scale and heavily penalised.
\end{itemize}
The goal is not to underfit richness, but to avoid allowing \(\alpha\) to become a universal explanation for all pairwise mismatch.

\subsection{Separating spatial and environmental structure}

Spatial structure is a particularly important source of confounding because both richness and compositional turnover are typically spatially autocorrelated. If flexible spatial residual structure is allowed in both the \(\alpha\) and \(\beta\) components, the two sub-models may compete to explain the same latent gradient. Identifiability is improved by assigning spatial structure more deliberately, for example by:
\begin{itemize}
    \item placing residual spatial structure in \(\alpha\) only and keeping \(\beta\) purely environmental,
    \item placing geographic separation in \(\beta\) only and keeping \(\alpha\) non-spatial,
    \item or restricting the two components to operate at different spatial scales (e.g.\ broad-scale residual structure in \(\alpha\), local environmental turnover in \(\beta\)).
\end{itemize}
In all cases, the preferred strategy is to avoid giving both sub-models unrestricted access to the same latent spatial signal.

\subsection{Observation-process covariates and auxiliary information}

Under the equal-draw assumption, \(\alpha\) should be interpreted as an \emph{effective richness under the recording process}, not necessarily a direct estimate of true species richness. This means that sampling effort, detectability, checklist completeness, and species-evenness effects can all enter the \(\alpha\) estimate. If these recording-process components are not modeled explicitly, they may be confounded with turnover.

As described in Section~\ref{sec:effort-decomposition}, the current implementation addresses this directly through an additive effort decomposition:
\begin{equation}
    \alpha_i = \text{softplus}\!\big(f_\alpha(\mathbf{z}_i) + f_\omega(\mathbf{w}_i)\big) + 1,
\end{equation}
where six spatial proxies for observation effort (ALA record density, institutional proximity, road access, etc.) are modeled through a dedicated EffortNet sub-network. This allows the environmental component $f_\alpha$ to focus on true ecological variation while the effort component $f_\omega$ absorbs recording-process artefacts. At prediction time, zeroing the effort logit recovers an estimate of richness driven purely by environmental conditions.

The additive structure on the logit scale means that effort acts as a multiplicative scaling of the match probability: sites with high recording effort will have higher apparent richness (and therefore lower within-site match probability), which is consistent with the intuition that better-sampled sites reveal more species under the equal-draw framework.

Where richer site-level biodiversity summaries are available, a further improvement is to include an auxiliary loss or penalty that encourages \(\alpha\) to remain consistent with externally estimated richness-like quantities.

\subsection{Pair design and training strategy (not yet implemented)}

Identifiability is also influenced by which pairs are seen during fitting. Pairs are most informative when they help break the correlation between geography, environment, and expected turnover. In particular, it is helpful to include:
\begin{itemize}
    \item pairs spanning small, moderate, and large environmental distances,
    \item geographically distant but environmentally similar pairs,
    \item geographically close but environmentally distinct pairs,
    \item and sufficient repeated within-site information to stabilise \(\alpha\).
\end{itemize}
From an optimisation perspective, staged fitting can further improve stability. A practical sequence is:
\begin{enumerate}
    \item fit \(\alpha\) primarily from within-site pairs and any richness constraints,
    \item fit \(\beta\) conditional on that \(\alpha\) baseline,
    \item jointly fine-tune both components,
    \item and introduce any spatial residual term only after the non-spatial model is stable.
\end{enumerate}
This does not change the statistical model, but helps preserve the intended functional roles of the two components during training.

\subsection{Summary}

In short, the \(\alpha\) and \(\beta\) components become more identifiable when they are forced to answer different questions from different evidence sources:
\begin{itemize}
    \item \(\alpha\) should primarily explain within-site matching and site-level richness/recording-process variation,
    \item \(\beta\) should primarily explain the decay in similarity with between-site environmental separation,
    \item and residual spatial or observational effects should be assigned deliberately rather than left for both components to absorb opportunistically.
\end{itemize}
These considerations are important regardless of whether the sub-models are specified using splines, monotone neural networks, or lattice-style architectures.

\section{Further thoughts}

\begin{itemize}
    \item All the interpretive considerations from the TMB document apply: $\alpha$ represents ``effective richness under the recording process'' rather than true species counts, and the equal-draw assumption means our estimates conflate richness with evenness.
    \item The additive monotone network can be interrogated post-hoc by sweeping each environmental dimension while holding others at zero, producing per-variable response curves analogous to the I-spline partial functions in GDM. The factored network additionally allows examination of joint response surfaces.
    \item The ability to capture variable interactions (factored beta) is a double-edged sword --- more flexible, but harder to interpret. Careful examination of joint response surfaces is recommended. The additive beta is preferred when interpretability is paramount.
    \item The effort/detectability decomposition (Section~\ref{sec:effort-decomposition}) provides a principled way to separate observation-process artefacts from ecological signal in the alpha model. The effort logit surface is a useful diagnostic: regions where $f_\omega > 0$ indicate that recording effort inflates apparent richness, while $f_\omega < 0$ indicates under-sampled areas. Comparing the environment-only and full alpha surfaces quantifies the magnitude of the sampling-effort correction.
    \item The network's richness predictions should be compared against the TMB P-spline predictions on the same data to assess consistency between the two approaches.
    \item If uncertainty quantification is needed (which the TMB \texttt{sdreport} provides), options include MC dropout at prediction time, deep ensembles, or training multiple models with different random seeds.
    \item The data pipeline (Steps 1--4 of the R pipeline) is shared between both approaches, so switching between TMB and NN fitting requires only an export step (\texttt{export\_for\_nn.R}) and a single Python command.
    \item The cyclic block-coordinate descent training (default mode) most closely mirrors the iterative R pipeline (\texttt{clesso\_iterative.R}), alternating alpha and beta optimisation until convergence.
    \item The two-phase cyclic fine-tuning procedure (\texttt{cyclic\_finetune}) provides a principled way to separate environmental signal from geographic distance decay, reducing the risk of geographic confounding during initial model fitting.
    \item The observed-richness lower-bound penalty should use \textit{corrected} $S_{\text{obs}}$ values (true species counts from raw data) rather than the subsampled pair-source richness, which underestimates true counts.
\end{itemize}
